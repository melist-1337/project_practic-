# 🥗 NutriPlaner Telegram Bot

## 📌 Описание проекта
**NutriPlaner** — это интеллектуальный асинхронный Telegram-бот, разработанный на языке Python с использованием фреймворка `aiogram 3.x`. Он предназначен для автоматизации сбора антропометрических данных, ведения персональной статистики изменений состава тела и гибкого расчёта макронутриентов (белков, жиров, углеводов) под конкретные фитнес-цели. 

Технология ориентирована на автоматизацию процессов контроля питания для пользователей, стремящихся к качественному изменению композиции тела (целевой диапазон: 10–12% жира).

### Ключевой функционал:
* **FSM (Finite State Machine):** Реализация пошагового диалогового сбора данных без потери контекста.
* **In-Memory Storage:** Использование легковесных структур динамической памяти для мгновенной обработки и изоляции профилей пользователей (`USER_DATA`).
* **Адаптивный алгоритм расчета:** Автоматическая конфигурация калорийности и БЖУ на основе сухой массы тела под три базовые фитнес-стратегии (Похудение, Рекомпозиция, Набор массы).
* **Логирование прогресса:** Ведение сквозной истории изменения массы тела с автоматическим расчетом дельты относительно предыдущего замера.

## 📚 Исследование предметной области

Разработка технологии велась на основе строго зафиксированной последовательности аналитических и проектировочных шагов:

### 1. Анализ фитнес-стратегий и физиологии композиции тела
На первом этапе были формализованы требования к диетическим режимам в зависимости от выбранного вектора развития. Были определены следующие закономерности:
* **Похудение:** Создание безопасного дефицита калорий (15–20% от уровня поддержания) при сохранении оптимального количества белка ($2.0$ г на кг веса) для минимизации катаболизма мышц.
* **Рекомпозиция:** Целевой режим для одновременного сжигания жира и удержания/роста мышечных волокон. Требует минимального дефицита (10%) и повышенного белкового порога ($2.2$ г на кг веса).
* **Набор массы:** Создание умеренного профицита энергии (10–15%) для обеспечения анаболических процессов в сочетании с силовыми тренировками.

### 2. Математическое моделирование расчетного ядра
Классические формулы (Маффина-Джеора, Харриса-Бенедикта) опираются на общую массу тела, что дает большую погрешность у людей с развитой мускулатурой или избыточным весом. В качестве ядра системы была выбрана **формула Кэтча-МакАрдла (Katch-McArdle)**, базирующаяся на количестве безжировой (сухой) ткани.

Математический аппарат технологии:
1. Вычисление сухой массы тела ($LBM$):
   $$LBM = W \times \left(1 - \frac{Fat\%}{100}\right)$$
   Где $W$ — общая масса тела (кг), $Fat\%$ — процент подкожного жира.
2. Вычисление базового обмена веществ ($BMR$):
   $$BMR = 370 + (21.6 \times LBM)$$
3. Определение суточной нормы расхода энергии с учетом минимального коэффициента активности ($Maintenance$):
   $$Maintenance = BMR \times 1.2$$

### 3. Исследование архитектуры хранения данных
Для обеспечения высокой скорости отклика в рамках MVP (минимально жизнеспособного продукта) был проведен сравнительный анализ подходов хранения данных. Стационарные реляционные СУБД (SQLite/PostgreSQL) были временно исключены для оптимизации развертывания. Выбор сделан в пользу Stateful-структуры (`In-memory storage`) на базе хэш-таблиц (Python Dictionary), привязанных к уникальному `user_id` мессенджера Telegram.
